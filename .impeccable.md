# TechDaily — Design Context

## Design Context

### Users
两类读者兼顾：
1. **快速扫读者**——工程师/开发者，碎片时间浏览，需要在30秒内判断一篇文章值不值得精读
2. **深度阅读者**——技术决策者/管理者，愿意花时间读完整分析，关注战略意义

核心任务：让两类读者都能在同一个页面上各取所需——扫读者得到"要不要读"的判断，深读者得到真正的洞察价值。

### Brand Personality
**三个词：精准、有格调、有观点**

- 保留编辑媒体的权威感（The Economist / The Atlantic 的严肃）
- 但要现代、精美、有设计创新感——不是复古报纸，是当代数字编辑媒体
- 有独特视角，不是中立聚合器，是"有态度的策展人"

### Aesthetic Direction
**暖色纸质感 × 现代精致**

- 底色：保留温暖的米白纸质感（#faf8f4 系），不换成深色主题
- 但对比度、层次要更丰富：不只是"浅色背景+深色文字"，要有明暗节奏
- 强调色：保留墨绿（forest green）作为权威/来源标识，保留琥珀（amber）作为"价值/行动"区
- 新增：允许用极深的墨色区块（#1e293b）做"关键洞察"的视觉锚点——形成强烈的明暗对比
- 字体排版：Playfair Display（大标题）+ Source Serif 4（正文）+ IBM Plex Mono（元数据）这套保留，但要更敢用大字号、更有节奏感

**参考感：** 介于 Stripe Press × The Atlantic × Monocle 数字版之间

**反参考：** 避免——平淡的卡片列表、AI 感十足的圆角玻璃拟态、过度使用分割线

### Design Principles

1. **层次即导航**：视觉层次本身就是读者的导航系统。核心摘要要最显眼，让人一眼抓住；关键要点要适合扫读；深度解读和洞察适合沉浸阅读。不同模块用不同的视觉形态，而不是统一的方块。

2. **留白是一等公民**：宁可内容少，不要版面挤。大标题需要呼吸空间，段落之间的节奏感比信息密度更重要。

3. **每个模块有自己的性格**：
   - 核心摘要 → 大号引言式，像杂志封面语句
   - 关键要点 → 紧凑清单，适合扫读
   - 深度解读 → 正文散文，像长篇报道
   - 关键洞察 → 深色反转区块，视觉上的"暂停时刻"，像印在黑板上的话
   - 为什么值得关注 → 琥珀暖色区，行动导向，但弱化为辅助信息

4. **排版决定品质**：字间距、行高、字重的细节区分高下。标题要有冲击力（大字号、紧字距），正文要舒适（宽行高），元数据要精致（小号等宽字体）。

5. **克制使用装饰**：装饰只服务于内容结构。左侧色条用来区分区块，分割线要轻，不能喧宾夺主。禁止渐变叠渐变、阴影叠阴影。

### Module Priority
按视觉权重排序（高→低）：
1. 核心摘要 — 最大、最显眼，是整篇的"钩子"
2. 关键洞察 — 深色反转块，视觉锚点，独家价值
3. 关键要点 — 清单形态，扫读友好
4. 深度解读 — 正文散文，沉浸阅读
5. 为什么值得关注 — 琥珀暖色，辅助收尾，视觉上弱化
